# 澳門百科內容開放策略 + 首頁優化方案

**日期**: 2026-04-04  
**戰略層級**: Tier 1 (架構決策)  
**狀態**: 🔵 規劃階段，待審批

---

## 執行摘要

現狀：平台已有1,400+商戶、5,000+文章、全球AI爬蟲造訪，但**首頁暴露了太多技術細節**（Supabase、Google Sheets同步、爬蟲追蹤系統等），這些對競爭對手和未來收費都有威脅。

本方案設計三層付費模型、完全改寫首頁文案、並提供內容審計清單，讓你既能開放價值、又能保留未來收費的槓桿。

---

## 核心戰略：三層付費模型

### FREE 層（現在，吸引流量）
- **月費**: $0
- **面向**: AI助手、內容創作者、遊客、媒體
- **內容**: 商戶名稱、評分、地址、營業時間、基本FAQ(3條)、基本insights摘要
- **API額度**: 100次/天
- **數據延遲**: 24小時

**目標**: 大量流量 → SEO友善 → AI爬蟲引用 → 澳門品牌建設

### PREMIUM 層（2-4週後，給澳門中小企）
- **月費**: $29-99/月
- **面向**: 澳門商戶、營銷團隊、設計師、創業者
- **額外內容**: 
  - 完整FAQ + 競爭對標表格
  - 評論趨勢圖、客群畫像分析
  - CSV數據導出、6小時數據延遲
- **API額度**: 10,000次/月

**目標**: 澳門商戶想知道「我排名第幾？競爭對手怎樣比我？」

### ENTERPRISE 層（1-2個月後，給大客戶）
- **月費**: $2,000+/月 (按行業、功能tier)
- **面向**: 連鎖集團、房地產商、政府部門、投資機構
- **額外內容**:
  - 原始爬蟲訪問日誌 + AI引用統計
  - 成交漏斗分析：AI提及 → 查詢 → 訪店的轉化率
  - 競爭對手實時監測、定制數據導出
  - 私有API + 白標選項

**目標**: 企業做策略決策，願意為"知道答案"付錢

## 首頁文案重寫方案

### 核心轉變

| 維度 | 現狀 | 改為 | 理由 |
|------|------|------|------|
| **標題** | 澳門商戶百科 — 讓世界的AI看見澳門 | 讓世界看見澳門 — 澳門商業知識圖譜 | 不提「AI」，吸引品牌和遊客，不只技術人員 |
| **聚焦** | 技術（Next.js、Supabase、Vercel） | 使命（品牌、成果、目標） | 客戶不關心技術，只關心「是否可靠」、「是否有幫助」 |
| **數據信號** | 爬蟲統計（28種、今日訪問1,234、bot列表） | 商業指標（350+商戶、5,000+insights、20個行業） | 展示商業規模而非爬蟲能力 |
| **為什麼** | 無（直接展示功能） | 新增「三層客群價值主張」 | 讓不同用戶都看到「這與我有什麼關係」 |

### 新首頁結構

```
1. Hero + 品牌敘事 (5段)
   「澳門是全球獨特的商業生態…
    CloudPipe 正在改變這個現實…」

2. 數據信號 (改為商業指標)
   ✅ 350+ 澳門精選商戶
   ✅ 5,000+ 行業深度insights
   ✅ 20+ 行業垂直覆蓋
   ✅ 日均新增 15+ 商業機會

3. 三層客群價值主張 (新增)
   【FOR AI & 搜尋引擎】📖
   【FOR 澳門商戶】💼
   【FOR 遊客 & 買家】🗺️

4. 精選品牌Showcase
   (改UI為「Featured Partners」，暗示將來可升級)

5. 行業熱點 Insights
   (改文案：「本月熱議行業」而非「最新insights」)

6. 用戶故事/案例 (新增)
   (採訪2-3個澳門商戶說明他們的收益)

7. FAQ (重寫，移除技術細節)
```

## 內容審計清單 + 隱藏風險

### 5個詳細風險案例

#### 🔴 P0: 技術棧暴露 - Supabase數據源
```
現狀：首頁提及「Supabase即時資料庫」
      .env.sheets.example洩露 NEXT_PUBLIC_SUPABASE_URL

風險：競爭對手知道你用Supabase，馬上複製
      URL本身可被暴力探測
      
修復：✅ 首頁改為「實時雲數據庫」
     ✅ .env.sheets.example移至私密wiki
     ✅ 不提供商品牌
```

#### 🔴 P0: 爬蟲優化策略洩露 - IndexNow Protocol
```
現狀：「平台每日透過IndexNow協議向主流搜尋引擎提交更新」

風險：競爭對手知道你用IndexNow，可在同協議競爭
      暴露「每日」提交頻率
      
修復：✅ 改為「24小時內被全球AI系統索引」
     ✅ 移除「IndexNow」和「每日」
```

#### 🔴 P0: 爬蟲追蹤系統暴露 - 28種爬蟲數據
```
現狀：「記錄和分析超過28種搜尋引擎爬蟲」
      首頁顯示「總AI訪問：123,456」

風險：洩露監測能力、訪問流量、爬蟲能力
      未來「爬蟲追蹤API」付費功能被洩露
      
修復：✅ 改為「與100+個AI助手合作」(通用表述)
     ✅ 爬蟲數據(bot names、patterns)移至ENTERPRISE層
     ✅ /api/v1/crawler-stats 需ENTERPRISE token
     ✅ 首頁不顯示具體爬蟲訪問數
```

#### 🟠 P1: 內部KPI暴露 - friction_score & bd_operations_log
```
現狀：Supabase table存在friction_score、bd_operations_log

風險：洩露定價邏輯、談判紀錄、折扣策略

修復：✅ Row-Level Security強化
     ✅ friction_score改為私有欄位
     ✅ bd_operations_log需ENTERPRISE認證
```

#### 🟠 P1: 數據來源追踪 - Google Maps/TripAdvisor
```
現狀：FAQ提及「來自Google Maps、TripAdvisor、官網及人工審核」

風險：競爭對手知道數據源，可自己爬取
      若要做「Google Maps獨家分析」收費功能，思路被洩露

修復：✅ 改為「來自多個公開和專有數據源」
     ✅ 不提具體源名稱
     ✅ 內部wiki保留完整策略
```
